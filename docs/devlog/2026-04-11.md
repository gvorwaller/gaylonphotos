### Key Technical Insight
- The pedigree tree re-rooting works by prefix-matching branchKeys (e.g., `root.m.f` for Self's mother's father), filtering to only descendants of that prefix, then remapping keys (`root.m.f` becomes `root`, `root.m.f.f` becomes `root.f`) and offsetting generations. All done client-side in the existing `nodes` derived.
- The Lightbox component is separate from PhotoDetail — new photo fields must be added to both rendering paths.
- Data directories are gitignored; prod data must be synced via rsync/scp separately from code deploys.
