# 2026-05-12 Devlog

## 13:31 EST - Family Heritage Default-On + Timeline Map Goto Button

### Summary
Two travel-collection UX improvements: the "Show Family Heritage" checkbox now defaults to checked on first visit to collections that have ancestry data, and every Timeline stop (main + side trip) gained a `📍 Map` button that pans the map to the stop's coordinates and enables the viewport photo filter. Closes td-503ee2.

### What Was Done

**1. Show Family Heritage default-on**
- Previously, the toggle defaulted to `false` whenever sessionStorage had no value, even for collections with ancestry. The user had to manually check the box on every fresh visit.
- Added an `initialAncestryToggle(slug, data)` helper that returns `true` when the collection has `ancestry.persons.length > 0` and no stored choice exists; otherwise honors the stored `'true'`/`'false'`.
- Wrapped the `$state(...)` initializer in `untrack()` to avoid a Svelte warning about capturing the reactive `data` prop at mount.
- Shipped as `6ccc586`.

**2. Timeline map goto button (td-503ee2)**
- Asked the user clarifying questions via AskUserQuestion (survey format). Decisions:
  - Scope: side trips **and** main stops
  - Filter: viewport filter (reuse existing `mapFilterActive`)
  - UI: small green `📍 Map` button matching `.goto-btn` style
  - Persistence: none (existing map-pos sessionStorage already restores back-nav state)
- Added `ongotolocation` prop to `Timeline.svelte` plus a `gotoStop(stop)` helper that emits `{ lat, lng, zoom: 12, _ts: Date.now() }`.
- Inserted the button into both the main-stop timeline header and the nested side-trip header.
- Wired the parent `+page.svelte` to set `gotoTarget = target; mapFilterActive = true` — pans the map and auto-enables the photo viewport filter in one tap.
- Smoke-tested in Chrome on Scandinavia 2023: 5 buttons rendered (4 main stops + 1 side trip). Clicking the side-trip button flipped "Filter by Map" → "Clear Map Filter" and the gallery narrowed from 982 → 45 photos.
- Shipped as `0ca2e36`.

### Files Modified

### Key Technical Insight
- Svelte 5 `$state(...)` initializers that reference `$props()` values trigger a `state_referenced_locally` warning because the prop is reactive but the initializer runs once. The fix is `$state(untrack(() => initFn(props)))` — same pattern the file already used for `_initSlug`.
- The viewport filter "just works" for the goto button because `ItineraryMap` fires `onboundschange` after the map settles, which updates `mapBounds`, which drives `displayPhotos` through `isInBounds`. No new filter plumbing required.
