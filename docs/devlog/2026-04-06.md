## 15:56 EST - Claude Code Bash Permissions Setup

### Summary
Configured bash command allow/deny lists in `~/.claude/settings.json` based on analysis of 1,307 commands across 38 session files from all projects.

### What Was Done
- Analyzed session history across all Claude Code projects (gaylonphotos, BTC-dashboard, giftlist, global) to identify frequently used bash commands
- Built a 67-pattern allow list covering: npm/node, git read ops + commit, td, gh CLI, deploy scripts, ssh/scp, curl, unix utils, python3/sqlite3/psql, macOS tools (osascript, swift), pm2
- Set deny list: `rm -rf *`, `git push *`, `git reset --hard *`, `git merge *`
- Removed `rm *` from allow list after review feedback — prefix-based pattern matching makes it risky even with deny precedence; individual `rm` commands will prompt instead
- Added `pm2 *` to allow list (heavily used for droplet process management, was missing from initial draft)

### Files Modified
- `~/.claude/settings.json` — added `permissions.allow` (67 patterns) and `permissions.deny` (4 patterns)

### Key Technical Insight
Claude Code's permission pattern matching is prefix-based with deny-first precedence. While `Bash(rm -rf *)` deny technically overrides `Bash(rm *)` allow, it's safer to omit the broad allow entirely for catastrophic commands — defense in depth over relying on rule ordering.
