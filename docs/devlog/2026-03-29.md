### Key Technical Insight
The compact occupied-branch layout was the key design choice. Instead of allocating `2^maxGen` vertical slots (which would be 256 rows at depth 8), the DFS leaf-first algorithm assigns sequential rows only to nodes that actually exist, then positions each parent at the midpoint of its children. This keeps the tree tight — at maxGen=5, ~117 nodes render in a compact vertical space instead of 32 empty leaf slots. The `branchKey` system (`root.f.m.f` = father's mother's father) provides a stable structural key for tree traversal without needing stored parent-child IDs.

### Key Technical Insight
When swapping person IDs in a relational dataset, you must also update all foreign key references. The migration correctly swapped `persons[].id` but initially missed the `places[].events[].personId` references — all 1129 event records became orphaned, causing empty tabs and broken detail cards. The fix was adding the same ID swap logic to the places events array. Lesson: always check for FK-like references when migrating IDs, even in schemaless JSON data.

  - Added `onmarkerclick` handler to `GeoTagger.svelte` that shows an InfoWindow with photo thumbnail, filename, GPS source, and a red "Clear GPS" button
  - "Clear GPS" calls `PUT /api/photos` to null out `gps`, `gpsSource`, and `locationName`, then moves the photo back to the untagged list for re-tagging
  - Used `window.__clearGeotagGps__` global callback pattern since Google Maps InfoWindow content is raw HTML (no Svelte binding)

  - Enabled `infoWindowEnabled` on the Map component

### Files Modified
- `src/lib/components/admin/AncestryEditor.svelte` (displayLineage helper + label)
- `src/lib/components/admin/GeoTagger.svelte` (marker click handler, clearGps, InfoWindow)

### Key Technical Insight
Google Maps InfoWindow content is injected as raw HTML strings, not Svelte components. To handle button clicks inside the InfoWindow, a global `window.__clearGeotagGps__` callback is registered that the inline `onclick` handler calls. This bridges the gap between Google Maps' DOM and Svelte's reactive state — the callback updates `localAllPhotos` and `untaggedPhotos` to trigger marker and list re-renders.
