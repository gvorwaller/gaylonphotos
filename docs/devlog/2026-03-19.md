### Key Technical Insight
- Google's browser API key (HTTP referrer restricted) cannot be used for server-side Geocoding API calls — need a separate key with "None" application restriction, limited to Geocoding API only
- For obscure historical/farm place names that no geocoder can resolve, cross-referencing a person's other known event locations is an effective research strategy — most people lived within a regional area, so their residence places cluster geographically
- The `PATCH` endpoint pattern (update one entity field) is much lighter than `PUT` (replace entire ancestry object) for interactive geo-tagging where you're updating one place at a time
