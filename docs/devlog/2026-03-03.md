### What Was Done

  - Prefixes new lineage labels (`wife-paternal`, `wife-maternal`, `wife-self`, `wife-both`)

### Key Technical Insight
The merge dedup strategy uses FamilySearch IDs (fsId) as the unique key. If husband and wife share common ancestors (unlikely but possible in endogamous populations), those persons won't be duplicated. The lineage prefix approach (`wife-paternal` vs `paternal`) keeps the two trees cleanly separated while sharing the same places array — place events from both trees appear together, sorted chronologically.

### Key Technical Insight
The hostile adversarial reviewer has an inherent bias toward finding issues — after 5 rounds, remaining P2/P3 flags are often false positives where the reviewer is being overzealous. The sanity validation step uses a fundamentally different agent persona (objective validator, not hostile) to filter these out and also catch regressions introduced by the fix cycle itself. This addresses the recurring frustration of unresolved low-priority items at the end of review cycles.
