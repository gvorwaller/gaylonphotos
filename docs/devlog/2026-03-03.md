### Key Technical Insight
The merge dedup strategy uses FamilySearch IDs (fsId) as the unique key. If husband and wife share common ancestors (unlikely but possible in endogamous populations), those persons won't be duplicated. The lineage prefix approach (`wife-paternal` vs `paternal`) keeps the two trees cleanly separated while sharing the same places array — place events from both trees appear together, sorted chronologically.
