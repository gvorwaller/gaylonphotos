# 2026-02-28 Devlog

## 00:15 EST - iOS Upload Fixes & Body Size Limit

### Summary
Debugged and fixed photo uploads failing on iPad with "Expected multipart form data" error. Root cause was SvelteKit adapter-node's default 512KB body size limit rejecting all photo uploads. Also added contenteditable paste support for iOS (though Browse Files via Photo Library is the better workflow).

### What Was Done
- **Diagnosed upload failure** — "Expected multipart form data" error on both paste and Browse Files uploads from iPad. Traced to SvelteKit adapter-node's default `BODY_SIZE_LIMIT` of 512KB
- **BODY_SIZE_LIMIT=0 broke login** — setting to `0` was interpreted as "zero bytes allowed" not "disabled", causing all POST bodies (including login JSON) to fail with "Invalid JSON"
- **Fixed with BODY_SIZE_LIMIT=Infinity** — correctly disables the limit; Nginx enforces 55MB upstream via `client_max_body_size`
- **Created ecosystem.config.cjs** — PM2 ecosystem file to persist `BODY_SIZE_LIMIT=Infinity` and `NODE_ENV=production` across restarts
- **Updated deploy script** — uses `pm2 restart ecosystem.config.cjs --update-env` instead of bare `pm2 restart gaylonphotos`
- **Contenteditable paste** — made upload drop zone `contenteditable` so iOS shows "Paste" on long-press, with `onbeforeinput` blocking text entry and clipboard File cloning to survive iOS data invalidation
- **Discovered paste strips EXIF** — iOS clipboard only copies pixel data, not EXIF metadata (GPS, camera settings, date). Browse Files → Photo Library is the correct iPad workflow since it preserves the original file

### Files Modified
- `ecosystem.config.cjs` (NEW) — PM2 config with env vars
- `scripts/deploy-to-DO.sh` — uses ecosystem config for restart
- `src/lib/components/admin/PhotoUploader.svelte` — contenteditable paste, iOS File cloning

### Key Technical Insight
- SvelteKit adapter-node `BODY_SIZE_LIMIT` defaults to 512KB — any file upload will fail silently with a generic formData parse error. Must set to `Infinity` (not `0`!) when Nginx handles limits upstream.
- `BODY_SIZE_LIMIT=0` is documented as "disabled" but behaves as "zero bytes" in practice — use `Infinity` instead.
- iOS clipboard paste strips all EXIF metadata — only raw pixel data is copied. For a photography app, Browse Files → Photo Library is the correct upload path.
- PM2 inline env vars (`VAR=x pm2 start`) work for the initial start but don't persist across `pm2 restart`. Use an ecosystem config file for durable env vars.
