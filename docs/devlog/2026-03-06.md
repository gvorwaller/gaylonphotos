## 01:04 EST - Bird Species Auto-ID via Vision AI

### Summary
Implemented auto-populate bird species feature using vision AI. Built three integration points: per-photo Auto-ID button in PhotoEditor, bulk Auto-ID on admin collection page with confirmation modal, and `--species` CLI flag on the ingest script. Initially built on Claude Haiku 4.5, then pivoted to OpenAI GPT-4.1-mini during review.

### What Was Done
- **Detailed implementation plan** — created `docs/species_id_tagging_V2.md` with step-by-step instructions covering all 8 files, exact insertion points, and verification steps
- **Shared prompt/config module** — `src/lib/vision-prompt.js` extracts `SPECIES_PROMPT`, `SPECIES_MODEL`, and `parseSpeciesResponse()` so both SvelteKit and script contexts share identical prompt logic
- **SvelteKit vision module** — `src/lib/server/vision.js` using OpenAI SDK with `$env/dynamic/private` for API key, lazy singleton client, null-on-error pattern
- **Standalone script module** — `scripts/vision-standalone.js` using `process.env.OPENAI_API_KEY`, accepts base64 buffer for pre-upload ingest context
- **API endpoint** — `src/routes/api/vision/+server.js` with collection type validation (wildlife-only), skip-if-already-identified logic, sequential processing, and batch cap of 50
- **PhotoEditor Auto-ID button** — inline button next to species input, AI confidence badge with scientific name, clears AI metadata when user manually edits species, disables Save during identification
- **Bulk Auto-ID** — header button with count, confirmation modal showing estimated cost (~$0.001/photo), real-time progress counter, cancellable via Stop button
- **Ingest script `--species` flag** — flag parsing, lazy vision import, species identification after display buffer generation, summary output

### Files Modified

- `.env` / `.env.example` — added `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`
- `src/lib/vision-prompt.js` — new shared prompt constants and response parser
- `src/lib/server/vision.js` — new SvelteKit vision module (OpenAI GPT-4.1-mini)
- `scripts/vision-standalone.js` — new standalone vision module for ingest
- `src/routes/api/vision/+server.js` — new POST endpoint with wildlife-only guard
- `src/lib/components/admin/PhotoEditor.svelte` — Auto-ID button, AI badge, species-row CSS, AI metadata clearing on manual edit
- `src/routes/admin/[collection]/+page.svelte` — bulk Auto-ID button, modal, progress, cancel support
- `scripts/ingest-photos.js` — `--species` flag, lazy vision import, species in photo object
- `docs/species_id_tagging_V2.md` — detailed implementation plan

### Key Technical Insight
- Pivoted from Anthropic Haiku to OpenAI GPT-4.1-mini during review — required switching from `@anthropic-ai/sdk` message format to OpenAI chat completions format (`image_url` type with data URLs for base64, `chat.completions.create` API)
- Extracted shared prompt into `src/lib/vision-prompt.js` to avoid drift between SvelteKit and script modules — both import the same `SPECIES_PROMPT`, `SPECIES_MODEL`, and `parseSpeciesResponse()`
- Used `$env/dynamic/private` instead of `$env/static/private` in the SvelteKit module — dynamic allows runtime key changes without rebuild
- API endpoint skips photos that already have species set (`photo.species` truthy check), preventing redundant API calls during bulk operations
- Ingest script uses `displayBuffer` (1600px) instead of `thumbBuffer` (400px) for better species identification accuracy
- PhotoEditor clears `scientificName` and `speciesAI` when user manually changes species, keeping AI metadata consistent

## 01:55 EST - Adversarial Review, Model Switch & Deploy

### Summary
Completed 3-round adversarial code review of the species auto-ID feature, switched from Claude Haiku to OpenAI GPT-4.1-mini after live testing revealed poor identification accuracy, fixed a critical image resolution issue, and deployed to production.

### What Was Done
- **Adversarial review (3 rounds + sanity validation)** — 16 total fixes across return value safety, JSON parsing robustness, re-entry guards, type validation, and UX improvements
- **Live browser testing via claude-in-chrome** — discovered Claude Haiku 4.5 misidentified a palm warbler as "Willow Flycatcher" / "Spotted Flycatcher" across multiple attempts
- **Model switch to OpenAI GPT-4.1-mini** — rewrote `src/lib/server/vision.js` and `scripts/vision-standalone.js` from Anthropic SDK to OpenAI SDK; updated shared model constant
- **Image resolution fix** — discovered 400px thumbnails caused misidentification across all models; switched vision API to use 1600px display images (`photo.url`) instead of thumbnails; all 4 tested OpenAI models (gpt-4.1-mini, gpt-4.1, gpt-4o, gpt-4o-mini) correctly identified the palm warbler with high confidence using the full image
- **Deploy fix** — `$env/static/private` failed build on droplet when `OPENAI_API_KEY` was missing; switched to `$env/dynamic/private` for graceful degradation
- **Production deploy** — added `OPENAI_API_KEY` to droplet `.env`, stashed dirty `package-lock.json` on server, deployed successfully (https://gaylon.photos/ returned 200), cleaned up stash

### Files Modified
- `src/lib/server/vision.js` — rewritten for OpenAI SDK, `$env/dynamic/private`, uses `imageUrl` param
- `scripts/vision-standalone.js` — rewritten for OpenAI SDK, base64 data URL approach
- `src/lib/vision-prompt.js` — model constant changed to `gpt-4.1-mini`
- `src/routes/api/vision/+server.js` — uses `photo.url` (1600px) instead of `photo.thumbnail` (400px)
- `scripts/ingest-photos.js` — uses `displayBuffer` instead of `thumbBuffer`, checks `OPENAI_API_KEY`
- `.env.example` — `OPENAI_API_KEY` replaces `ANTHROPIC_API_KEY`

### Key Technical Insight
- **Thumbnail resolution kills bird ID accuracy** — GPT-4.1-mini identified a palm warbler as "Dusky Warbler" from the 400px thumbnail but correctly as "Palm Warbler" (high confidence) from the 1600px display image. All four OpenAI vision models showed the same pattern. The extra ~$0.001/photo for larger images is well worth it.
- **`$env/static/private` vs `$env/dynamic/private`** — static imports fail the entire build if the variable is missing from `.env`. Dynamic imports read at runtime and allow graceful fallback. Use dynamic for optional features like vision AI.
- **Droplet drift** — `npm install` on the server can modify `package-lock.json` differently than local, causing merge conflicts on deploy. The deploy script should probably include `git checkout -- package-lock.json` or `git stash` as a standard step.
