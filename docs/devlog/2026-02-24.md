### Key Technical Insight
- **Svelte component imports can shadow JS builtins.** `import Map from '...Map.svelte'` is a module-level default import that replaces `globalThis.Map` in the entire module scope. During SSR, `new Map()` then tries to instantiate the Svelte component (which is a function, not a Map constructor), producing `TypeError: Cannot read properties of undefined (reading 'component')`. Fix: always use non-colliding names for component imports (`GoogleMap`, `MapView`, etc.).
- **SvelteKit 2.x `error()` already throws.** The function signature returns `never` and internally does `throw new HttpError(status, body)`. Adding `throw error()` is redundant but harmless ŌĆö it helps readability and TypeScript type narrowing. CODEX flagged this as P1 but it's not a runtime bug in SvelteKit 2.x.
