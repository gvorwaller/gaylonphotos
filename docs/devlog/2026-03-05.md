# 2026-03-05 Devlog

## 19:51 EST - Itinerary Admin Improvements + Data Backup Strategy

### Summary
Added Google Places search to the itinerary editor, fixed the polyline display bug caused by falsy 0-coord checks, gitignored runtime data files, and set up nightly production data backups via CCC preflight script.

### What Was Done
- **Google Places Autocomplete in ItineraryEditor** — added search bar above the map (same pattern as GeoTagger). Typing a place name pans/zooms the map; if a stop is in "pick GPS" mode, selecting a place auto-assigns coordinates to the active stop.
- **Fixed 0-coord polyline bug** — changed `s.lat && s.lng` (falsy check, excludes `0`) to `s.lat != null && s.lng != null && (s.lat !== 0 || s.lng !== 0)` in both ItineraryEditor (admin) and ItineraryMap (public). New stops default to `lat: 0, lng: 0`, so the old check filtered them out, preventing polyline rendering.
- **Gitignored `data/` directory** — removed all JSON data files from git tracking. Runtime data is managed via admin UI, not git. Prevents merge conflicts on deploy when server data diverges from repo.
- **Production data backup** — created `scripts/sync-prod-data.sh` that rsyncs DO droplet's `data/` to local `data-prod/`. Configured as CCC preflight script on the existing gaylonphotos backup task. Nightly at 03:00, CCC pulls prod data from DO then backs up everything (including `data-prod/`) to Synology NAS.
- **Deploy script fix** — server had local data file changes that conflicted with `git pull`. Resolved by stashing, pulling, restoring. This was the motivation for gitignoring `data/`.

### Key Technical Insight
- CCC runs preflight scripts in a minimal environment — no SSH agent, no PATH to homebrew. Scripts need full paths (`/opt/homebrew/bin/rsync`, `/usr/bin/ssh -i $HOME/.ssh/id_ed25519`) and `exit 0` to avoid aborting the backup task.
- The `data/` gitignore required careful deploy coordination: `git pull` deletes untracked-then-removed files from the working directory. Had to back up server data before pulling, then restore after.
- Photos live on DO Spaces (S3-compatible object storage with built-in redundancy) — only JSON metadata files need backup from the droplet.
