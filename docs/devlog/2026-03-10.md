### What Was Done

- **Bulk reverse-geocode script**: Created `scripts/bulk-reverse-geocode.js` to backfill `locationName` on existing photos. ### Key Technical Insight
- **HEIC on Linux servers**: sharp's prebuilt binaries don't include HEIC/HEVC decoders (licensing). Building from source requires libvips 8.17.3+ but Ubuntu 24.04 ships 8.15.1. The clean solution is client-side conversion — macOS has native HEIC support via ImageIO, so convert before upload
- **Google Maps API key restrictions**: Keys with HTTP referrer restrictions (for browser use) cannot be used with the Geocoding REST API. Either create a separate unrestricted key or use Nominatim as fallback. Nominatim quality is good for cities/towns, just slower (1 req/sec)
- **PM2 restart kills sessions**: In-memory session Map is cleared on restart. Mid-upload PM2 restart caused 291 auth failures. The retry mechanism handled this cleanly

## 15:45 EST - Bird Auto-ID Fix, Species Research, Geotag UI Overhaul

### Summary
Fixed bird auto-ID that was silently failing on production (0/56 identified), ran it successfully (34 high-confidence IDs), researched alternative AI for bird identification, and overhauled the geotag page layout so photos are actually visible.

### What Was Done
- **Diagnosed auto-ID failure**: Server logs showed `Vision: OPENAI_API_KEY not configured` for every attempt. Root cause: `vision.js` used `$env/dynamic/private` (reads `process.env` at runtime) but PM2 doesn't load `.env` files. Every other server module uses `$env/static/private` (baked in at build time). One-line import fix.
- **Ran auto-ID on 56 bird photos**: 34 identified with high confidence, 0 errors. Species include Laughing Gull, Osprey, Roseate Spoonbill, Palm Warbler, American Bittern, Painted Bunting, Ruby-throated Hummingbird, Limpkin, Black Swan, American Avocet, and more.
- **Filtered out medium-confidence IDs**: User confirmed medium-confidence results were incorrect. Changed `/api/vision` to only save high-confidence identifications. Cleared 8 incorrect medium-confidence IDs from production.
- **Researched bird ID alternatives**: Cornell's Merlin (best, no API), iNaturalist CV (excellent, no public API), Google Cloud Vision (poor for species), Gemini 2.0 Flash (good, 50x cheaper than GPT-4.1-mini at ~$0.00006/image). Recommendation: switch to Gemini.
- **Created td-c71ffe**: Switch bird auto-ID from GPT-4.1-mini to Gemini 2.0 Flash (P2)
- **Overhauled geotag page layout**: Original showed tiny 3-column square-cropped grid — photos were unidentifiable. Tried several approaches (2-col grid, aspect-ratio boxes, div vs button elements). Final solution: single-column card list with 120x80px thumbnail + species/filename label. Photos now clearly visible for geo-tagging.
- **Synced production data locally**: Copied `birds/photos.json` (56 photos) and `scandinavia-2023/photos.json` (977 photos) from production to local dev for testing.

### Files Modified
- `src/lib/server/vision.js` — switched from `$env/dynamic/private` to `$env/static/private` for OPENAI_API_KEY
- `src/routes/api/vision/+server.js` — skip medium/low confidence results, show rejected species in skip reason
- `src/lib/components/admin/GeoTagger.svelte` — replaced grid layout with card list (120x80 thumbnails, species labels)

- `data/birds/photos.json` — synced from production (56 photos with species data)
- `data/scandinavia-2023/photos.json` — synced from production (977 photos)

### Key Technical Insight
- **`$env/dynamic/private` vs `$env/static/private`**: SvelteKit's dynamic env reads from `process.env` at runtime — useless when PM2 doesn't load `.env`. Static env bakes values into the build at `npm run build` time, which is how all other server modules work. This silently broke bird auto-ID on production with zero errors in the API response (just "Could not identify" for every photo).
- **`<button>` elements in CSS Grid**: Buttons don't auto-size to content height in CSS Grid/Flexbox like `<div>` does. A button with `overflow: hidden` collapsed to 15px despite containing a 147px image. Switching to a card list layout with explicit `img` dimensions solved this cleanly.
- **Cornell's Merlin has no API**: The best bird ID model in the world (6,000+ species, trained on millions of eBird photos) is locked inside the Merlin mobile app. No REST API, no model download, no developer access.
