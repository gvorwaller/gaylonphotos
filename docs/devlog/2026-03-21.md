# 2026-03-21 Devlog

## 18:20 EST - Phase 4b + Phase 5: Map Previews, Photo Nav & Show on Map

### Summary
Implemented map marker click previews for wildlife/action collections, plus full photo navigation UX (counter, swipe, show on map) in both Lightbox and PhotoDetail views. Committed as `1840509` and deployed.

### What Was Done
- **4b — Map marker InfoWindows**: Wildlife markers show photo thumbnail, species, location, date. Action/surf markers show spot name, first photo, photo count + scroll-to-section.
- **5a — Photo counter**: "X of N" pill badge in both Lightbox (gallery overlay) and PhotoDetail (`/photo/[id]` page).
- **5b — Swipe navigation**: Arrow keys, touch swipe (60px threshold), and trackpad horizontal swipe (debounced deltaX accumulation, 80px threshold) in both viewing experiences.
- **5c — Show on Map**: "Show on Map →" link for GPS-tagged photos. In Lightbox, navigates to `?mapLat=&mapLng=` on the collection page. In PhotoDetail, same pattern via sidebar link. Collection page reads query params via `$effect` and sets `gotoTarget` to pan/zoom the map.
- **`collectionSlug` threading**: Passed through Gallery → Lightbox so map links work from gallery-opened lightbox.
- **`photoNav` server data**: Added lightweight navigation array (id + thumbnail) to photo detail server loader.

### Key Technical Insight
The app has two distinct photo viewing paths: Lightbox (full-screen overlay from gallery click) and PhotoDetail (dedicated `/photo/[id]` page). Features must be implemented in both. Cross-page map zoom uses URL query params (`?mapLat=&mapLng=`) read by a `$effect` on the collection page, setting a reactive `gotoTarget` that threads down to the Map component.
