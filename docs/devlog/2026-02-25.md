## 00:49 EST - Adversarial Code Review: Admin Setup & Auth Hardening

### Summary
Ran a 5-round hostile adversarial code review on uncommitted changes (bcryptjs migration, admin setup flow, auth improvements). Found and fixed 10+ bugs across security, correctness, and hardening categories before any commit.

### What Was Done
- **TOCTOU race in admin creation**: `createAdmin()` had a check-then-write race where concurrent `/api/auth/setup` requests could overwrite each other. Added `createJsonIfNotExists()` to `json-store.js` with per-file locking across the existence check and write.
- **CSRF bypass on `/api/auth/setup`**: The hooks exemption `startsWith('/api/auth')` let the setup endpoint skip CSRF origin validation. Restructured hooks to apply CSRF checks to ALL API mutations, with only the session-auth check exempted for auth endpoints.
- **Timing oracle on username**: `verifyCredentials()` returned immediately on username mismatch (fast) vs running bcrypt on password mismatch (slow), leaking whether a username exists. Now always runs `bcrypt.compare` against a dummy hash when username doesn't match.
- **Type guards on API inputs**: Both `/api/auth` and `/api/auth/setup` now validate `typeof` for all string inputs, preventing 500 errors from non-string payloads.
- **Session memory management**: Added periodic sweep (15min interval) and MAX_SESSIONS=100 cap with oldest-first eviction to prevent unbounded Map growth.
- **Stale `adminExists` recovery**: Login page now calls `invalidateAll()` when server returns "Admin already configured", transitioning the UI from setup form to login form without requiring a page reload.
- **Species filter reset**: Fixed bug where `filterSpecies` state persisted across collection navigations due to SvelteKit component reuse. Added `$effect` tracking `data.collection.slug`.
- **Admin layout data leak**: Guarded `listCollections()` behind `locals.user` check so unauthenticated visitors to `/admin/login` don't receive collection data in the SSR payload.
- **Misc fixes**: Login username trim, exact-match login route exemption, `sameSite: 'Lax'` casing, password field clearing on error, removed unused `node-addon-api`/`node-gyp` devDeps, `updateJson` undefined return guard, admin nav link hidden from public.

### Files Modified
- `src/lib/server/json-store.js` — `createJsonIfNotExists()`, ENOENT guard, undefined updater guard

- `src/hooks.server.js` — CSRF for all mutations, exact login exemption

### Key Technical Insight
The adversarial review loop is highly effective at catching cross-cutting bugs that single-pass reviews miss. The TOCTOU race in `createAdmin` was found in round 1, but the fix (`createJsonIfNotExists`) had its own bug (swallowing non-ENOENT errors) caught in round 2. The timing oracle on username comparison wasn't caught until round 5 — each round's fresh perspective found new classes of issues. The stale species filter bug (round 4) is a classic SvelteKit gotcha: component-level `$state` persists across same-route navigations because SvelteKit reuses component instances.

### Key Technical Insight
Cloudflare SSL mode matters when proxying. "Full" mode tries to connect to the origin over HTTPS (port 443), which fails with a 521 if Nginx only listens on port 80.
