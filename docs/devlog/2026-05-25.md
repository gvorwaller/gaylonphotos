# 2026-05-25 Devlog

## 23:06 EST - Google Maps API Key / Billing Fix

### Summary
Google Maps stopped loading on all travel collection pages ("This page can't load Google Maps correctly" + "For development purposes only" watermarks). Root cause was the Google Cloud free trial expiring, which closed the billing account and orphaned the API key.

### What Was Done

1. **Diagnosed via Google Cloud Console** — navigated to Billing and found "This project has no billing account." The free trial had ended, closing the billing account (`0135F0-C12F65-FA1F43`).
2. **Activated full billing account** — clicked Upgrade on the billing overview, converting from expired free trial to pay-as-you-go. Mastercard on file was still valid.
3. **Re-linked billing to project** — the `gaylonphotos` project had been unlinked when the trial expired. Linked it back to "My Billing Account."
4. **Maps JavaScript API re-enabled** — enabling the API triggered the Maps Platform onboarding flow, which generated a new API key (`REDACTED_GOOGLE_MAPS_KEY`). The old key (`AIzaSyBvKpx...`) was gone.
5. **Updated `.env` on local and production** — replaced the old key in both places.
6. **Deployed** — ran `deploy-to-DO.sh`. Initial manual `npm run build` via SSH failed because `node_modules` was incomplete (only 75 packages instead of 224). The deploy script ran `npm install` first, fixing it.

### Files Modified
- `.env` (local) — new `PUBLIC_GOOGLE_MAPS_API_KEY`
- `/opt/gaylonphotos/.env` (production) — same key update

### Key Technical Insight
- Google Cloud free trial expiration silently closes the billing account and unlinks it from projects. APIs that require billing (like Maps JavaScript API) degrade to "development purposes only" mode with watermarks rather than failing outright.
- The old API key was deleted when the trial ended — not just disabled. "Restore deleted credentials" showed nothing.
- Production `node_modules` was in a broken state (75 of 224 packages). The deploy script's `npm install` step fixed it, reinforcing why `deploy-to-DO.sh` should always be used instead of manual SSH builds.

## 23:09 EST - API Key Referrer Restriction

### Summary
Added HTTP referrer restriction to the new Maps Platform API key so it only accepts requests from `gaylon.photos`.

### What Was Done
- Navigated to Google Cloud Console → APIs & Services → Credentials → Maps Platform API Key
- Changed Application restrictions from "None" to "Websites"
- Added `gaylon.photos/*` as the allowed referrer
- Saved — takes up to 5 minutes for Google to propagate

### Files Modified
- None (Google Cloud Console change only)

### Key Technical Insight
- Without referrer restrictions, anyone who views the page source can copy the API key and use it on their own site, running up your Maps bill. The restriction ensures only requests originating from `gaylon.photos` are accepted.

### Next Steps
- Consider setting a budget alert in GCP billing to catch future billing issues early.
